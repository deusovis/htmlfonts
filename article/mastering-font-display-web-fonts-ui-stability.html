Mastering `font-display` for Flawless Web Font Loading & UI Stability

Optimize font loading by strategically choosing `font-display` values. For most body text, `swap` ensures quick content visibility. For critical brand fonts or icons, consider `fallback` or `optional` combined with `` to minimize render-blocking and avoid flashes of unstyled text (FOUT) or invisible text (FOIT), boosting perceived performance and UI stability.

/* For general text, prioritizing content visibility */
@font-face {
  font-family: 'Inter Sans';
  src: url('/fonts/Inter-Regular.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
  font-display: swap; /* Provides a fallback font immediately, swaps when web font loads */
}

/* For brand-critical headlines, prioritizing font aesthetic but with a short block period */
@font-face {
  font-family: 'Brand Display';
  src: url('/fonts/BrandDisplay-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: fallback; /* Short block, then swap if not loaded; avoids extreme FOUT */
}

/* For non-essential fonts (e.g., specific icons), prioritizing no layout shift */
@font-face {
  font-family: 'Icon Set';
  src: url('/fonts/IconSet.woff2') format('woff2');
  font-weight: normal;
  font-style: normal;
  font-display: optional; /* Uses the font only if it's available without causing a layout shift */
}
