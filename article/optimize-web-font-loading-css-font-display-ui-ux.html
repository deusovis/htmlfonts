Optimizing Web Font Loading for Flawless UI & User Experience with CSS `font-display`

Leverage `font-display: optional` for the best perceived performance on non-critical brand fonts. While `swap` quickly displays fallback text then swaps to the custom font (potentially causing a layout shift), `optional` will only use the custom font if it's available immediately or from cache. Otherwise, it sticks with the system font, preventing layout shifts and improving Cumulative Layout Shift (CLS) scores, crucial for a smooth user experience.

@font-face {
  font-family: 'Custom Brand Font';
  src: url('/fonts/CustomBrandFont-Regular.woff2') format('woff2'),
       url('/fonts/CustomBrandFont-Regular.woff') format('woff');
  font-weight: 400;
  font-style: normal;
  /* 'optional' for critical UI elements, avoids FOUT/FOIT and layout shift */
  font-display: optional; 
}

@font-face {
  font-family: 'Body Text Font';
  src: url('/fonts/BodyTextFont-Regular.woff2') format('woff2'),
       url('/fonts/BodyTextFont-Regular.woff') format('woff');
  font-weight: 400;
  font-style: normal;
  /* 'swap' is generally good for body text to prioritize readability */
  font-display: swap;
}

body {
  font-family: 'Body Text Font', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol';
  font-size: 1.125rem;
  line-height: 1.6;
  color: #222;
  text-rendering: optimizeLegibility;
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

h1 {
  font-family: 'Custom Brand Font', system-ui, sans-serif;
  font-size: 3rem;
  font-weight: 700;
  line-height: 1.1;
  letter-spacing: -0.03em;
}
